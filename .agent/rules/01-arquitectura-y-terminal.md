# Reglas de Arquitectura y Terminal
1. Todo el código frontend se aloja exclusivamente en `/src`. Los documentos de contexto o manuales de marca van en `/docs/context`.
2. Si ejecutas un comando de terminal (ej. `npm run dev` o `npm install`) y falla, NO te detengas a pedir ayuda inmediatamente. Lee el error (ej. puerto ocupado, dependencias faltantes), explícale al usuario qué falló brevemente y propón la solución.
